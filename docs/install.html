About Quipucords

The Quipucords tool is designed to help users collect data about their usage of specific Red Hat software. By using Quipucords, users can reduce the amount of time and effort that is required to calculate and report usage of those Red Hat products.

Learn more

To learn more about the purpose, benefits, and characteristics of Quipucords, see the following information:

  • What is Quipucords?

To learn more about the products and product versions that Quipucords can find and inspect, see the following information:

  • What products does Quipucords find?

What is Quipucords?

The Quipucords tool, also known as Quipucords, is an inspection and reporting tool. It is designed to find, identify, and report environment data, or facts, such as the number of physical and virtual systems on a network, their operating systems, and other configuration data. In addition, it is designed to find, identify, and report more detailed facts for some versions of key Red Hat packages and products for the IT resources in that network.

The ability to inspect the software and systems that are running on your network improves your ability to understand and report on your entitlement usage. Ultimately, this inspection and reporting process is part of the larger system administration task of managing your inventories.

The Quipucords tool requires the configuration of two basic structures to access IT resources and run the inspection process. A credential contains user access data, such as the user name and password or SSH key of a user with sufficient authority to run the inspection process on a particular source or some of the assets on that source. A source contains data about a single asset or multiple assets that are to be inspected. These assets can be physical machines, virtual machines, or containers, identified as host names, IP addresses, IP ranges, or subnets. These assets can also be a systems management solution such as vCenter Server or Red Hat Satellite Server.

You can save multiple credentials and sources to use with Quipucords in various combinations as you run inspection processes, or scans. When you have completed a scan, you can access these facts in the output as a collection of formatted data, or report, to review the results.

By default, the credentials and sources that are created during the use of Quipucords are encrypted in a database. The values are encrypted with AES-256 encryption. They are decrypted when the Quipucords server runs a scan with the use of a vault password to access the encrypted values that are stored in the database.

The Quipucords tool is an agentless inspection tool, so there is no need to install the tool on every source that is to be inspected. However, the system that Quipucords is installed on must have access to the systems to be discovered and inspected.

The Quipucords tool finds the following Red Hat products. For each version or release, the earliest version is listed, with later releases indicated as applicable.

If a product has changed names recently so that you might be more familiar with the current name for that product, that name is provided as additional information. No later version is implied by the inclusion of a newer product name unless specific versions of that product are also listed.

Installing prerequisites for Quipucords

Before you begin the installation process, review the information about Quipucords prerequisites. Then complete any installation or configuration tasks for the prerequisites.

Procedure

Install the following requirements for hardware, software, and the environment in which you are going to install and use Quipucords.

Hardware prerequisites

The system on which you are going to install Quipucords must meet or exceed the following hardware requirements:

  • CPU: 2 core minimum, with a recommended 4 cores

  • Disk Storage: 30 GB

  • RAM: 1 GB minimum, with a recommended 2 GB

Software prerequisites

The system on which you are going to install Quipucords must have the following software requirements installed:

  • Operating system: The latest release of the following operating system versions:

    • Red Hat Enterprise Linux 6, 7, or 8

    • CentOS 6 or 7

      Note

      RHEL 7 or 8 or CentOS 7 are the recommended operating system versions. For RHEL 6 or CentOS 6, the minimum required kernel is 2.6.32-431 or later.

  • Database:

    • PostgreSQL 9.6.10

      Note

      The database is automatically installed for you if you use the online installation.

  • Programming language for the Quipucords command line interface (CLI):

    • Python 3.4 or later

Other environment prerequisites

The environment in which you are going to use Quipucords must meet the following requirements. Some of these requirements affect the systems on which you are going to run Quipucords. Others affect the systems in your IT infrastructure that you are going to scan with Quipucords.

On the machine where Quipucords is installed:

  • The machine that Quipucords is installed on must have access to the IT infrastructure assets that are to be scanned.

  • If you install the Quipucords server and command line interface (CLI) tool on different machines, the CLI machine must have must have access to the IT infrastructure assets that are to be scanned and must be able to connect to the server machine.

On the systems where Quipucords runs scans:

  • Any network sources that are targeted for scanning must be running the Secure Shell (SSH) protocol.

  • A user account that is used as credential for a scan requires the bash shell. The shell cannot be the /sbin/nologin shell or the /bin/false shell.

  • A user account that is used as a credential for a network scan must have adequate permissions to run commands and read certain files on those systems. For example, some commands that run during a scan require privilege elevation to gather the complete set of facts for the scan. For more information about these commands, see "Commands that are used in scans of remote network assets" in Using Quipucords.

Installing Quipucords with the online installation process

The online, or automated, installation process runs an installer that uses Ansible to install the command line interface tool, the server image, and the database image.

When you run the online installation process, the server is installed and started as described in (…​.link to the default config description topic from Configuring and Starting Quipucords…​). However, you can change some of the defaults used by the automated installation with the (…​.link to Installation Options topic…​.).

Prerequisites

  • Ensure that all prerequisites are installed and configured.

  • To use the online installation process, the machine on which you are downloading the installer and installing Quipucords must be connected to the internet.

Procedure

To install Quipucords with the online installation process, you do the following tasks:

  1. Download the installer. For more information, see Downloading the installer.

  2. Run the installation. For more information, see Installing with the online installation process.

Downloading the installer

Procedure

  1. Download the Quipucords installer by using the following command, based on your operating system version:

    • For RHEL 8, use the following command:

      # yum install -y https://github.com/quipucords/qpc-tools/releases/latest/download/qpc_tools.el8.noarch.rpm

    • For RHEL 7 and CentOS 7, use the following command:

      # yum install -y https://github.com/quipucords/qpc-tools/releases/latest/download/qpc_tools.el7.noarch.rpm

    • For RHEL 6 and CentOS 6, use the following command:

      # yum install -y https://github.com/quipucords/quipucords-installer/releases/latest/download/quipucords_installer.el6.noarch.rpm

Installing with the online installation process

*CHANGING with change to installer to prompt for passwords (BUT some of this text will still be needed if user NOT prompted for user ID or email)* The simplest method for installing Quipucords is to run the online installation with all of the default options. However, accepting all of the default options does not change the default administrator user name and password for Quipucords, a process that does not meet common recommended security practices. In addition, the default server administrator email address for Quipucords is set to an example.com email domain, a process that does not provide the proper contact information for your organization.

Therefore, the recommended method is to run the online installation with the options that change the default administrator user name and password for Quipucords and change the email address for that primary administrator, or server user, to one that is appropriate for your organization. The following information describes this method.

Prerequisites

  • Download the Quipucords installer. For more information, see (…​link to download inst topic…​).

Procedure

*Command syntax for options changing…​*

  1. Start the installation by entering the following command, where admin is the new administrator user name, qpcpassw0rd is the new password, and admin@example.com is the email address for the Quipucords server user:

    # qpc-tools install -e server_username=admin -e server_password=qpcpassw0rd -e server_user_email=admin@example.com

Installing Quipucords with the offline installation process

The offline installation process is required when you install Quipucords in an offline environment, also commonly referred to as a disconnected or air-gapped environment. If you must install the server, command line interface (CLI) tool, or both on machines that do not have internet access, then you must use the offline installation process.

For the offline installation process, you must complete some steps from a connected machine, one that has internet connectivity. These steps include downloading the installer, downloading the server and CLI packages, building a container image for the database, transferring packages to the disconnected machine, and others. You also complete steps on the disconnected machine, including installing dependencies to enable the offline installation, receiving the prepared packages and images transferred from the connected machine, and running the installer itself. The dependencies that enable offline installation are in addition to the standard prerequisites.

When you run the offline installation process, the server is installed and started as described in (…​.link to the default config description topic from Configuring and Starting Quipucords…​). However, you can change some of the defaults used by the automated installation with the (…​.link to Installation Options topic…​.).

Prerequisites

  • Ensure that all prerequisites are installed and configured.

  • Ensure that the connected machine where you download packages can connect to the disconnected machine or machines where you want to install the Quipucords server and CLI tool.

Procedure

To install Quipucords with the offline installation process, you do the following tasks:

  1. Install dependencies. These dependencies are additional prerequisites that cannot be installed by the installer in an offline installation process. For more information, see Installing dependencies for the offline installation.

    Note

    These dependencies are required on both the connected machine where you are obtaining the packages and the disconnected machine where you are installing Quipucords.

  2. Download the installer. For more information, see Downloading the installer.

  3. *Maybe new step with newest installer changes—​create a dir on the connected machine?*

  4. Download the server package to the connected machine. For more information, see Downloading the server package.

  5. Build the database container image. For more information, see Building the database container image.

  6. Download the CLI package. For more information, see Downloading the command line interface package.

  7. Transfer the packages from the connected machine where they were downloaded to the disconnected machine where you are going to run the installation. For more information, see Transferring packages to the offline machine.

  8. Run the installation. For more information, see Installing with the offline installation process.

Installing dependencies for the offline installation

To use the offline installation process, you must install some dependencies manually. These dependencies are in addition to the prerequisites for Quipucords installation.

You must install the dependencies on the disconnected machine where you are installing and using Quipucords.

You must also install most of the dependencies, such as Ansible, the correct Python version, and the applicable container software, on the connected machine where you are downloading the packages before installation. For example, the yum install command that you use to get the Quipucords qpc-tools package also does an RPM package check for certain runtime requirements. This package check checks for the Ansible and Python dependencies. In addition, the container software (either Podman or Docker, depending on your RHEL or CentOS version) is required to be installed on the connected machine to build a container for the PostgreSQL database.

Procedure

Use one of the following procedures, as appropriate for your operating system.

Installing dependencies for RHEL 7 and 8 or CentOS 7 and 8

To install the dependencies on RHEL 7 and 8 or CentOS 7 and 8, use the following steps:

Dependencies for the Quipucords server package

  1. On both the connected and disconnected machine: Install Ansible 2.4 or later, depending on the requirements of your operating system version. For more information, see the documentation about installing the control node (also known as the control manager in previous versions) in the Ansible Installation Guide. You can find a link to the Installation Guide for more current versions plus a link to previous documentation versions at https://docs.ansible.com/#coreversionselect.

  2. On both the connected and disconnected machine: Install the latest version of Podman. For more information, see the documentation about installation located in the Getting Started with Podman guide, at https://podman.io/getting-started/.

Dependencies for the Quipucords CLI package

  1. On the disconnected machine only: Install the Extra Packages for Enterprise Linux (EPEL) package. Select the package that is appropriate for your operating system from the options in the "Quickstart" section at the following link: https://fedoraproject.org/wiki/EPEL.

  1. On both the connected and disconnected machine: Make sure that you are running a minimum of Python 3.6. In addition, make sure that the Python 3.6-requests module is installed.

Installing dependencies for RHEL 6 and CentOS 6

To install the dependencies on RHEL 6 and CentOS 6, use the following steps:

Dependencies for the Quipucords server package

  1. On both the connected and disconnected machine: Install Ansible 2.4 or later, depending on the requirements of your operating system version. For more information, see the documentation about installing the control node (also known as the control manager in previous versions) in the Ansible Installation Guide. You can find a link to the Installation Guide for more current versions plus a link to previous documentation versions at https://docs.ansible.com/#coreversionselect.

  2. On both the connected and disconnected machine: Install Docker. An older version of Docker that is compatible with RHEL 6 and CentOS 6 is required. This version of Docker is no longer formally supported. To download and install this version of Docker, select the /docker-engine-1.7.1-1.el6.x86_64.rpm package at the following link: https://yum.dockerproject.org/repo/main/centos/6/Packages/.

Dependencies for the Quipucords CLI package

  1. On the disconnected machine only: Install the Extra Packages for Enterprise Linux (EPEL) package. Select the package that is appropriate for your operating system from the options in the "Quickstart" section at the following link: https://fedoraproject.org/wiki/EPEL.

  1. On both the connected and disconnected machine: Make sure that you are running a minimum of Python 3.4. In addition, make sure that the Python 3.4-requests module is installed.

Downloading the server package

The Quipucords server package contains the server, database, and graphical user interface (GUI) client.

The server package is available as a container image. Download the server package to the connected machine.

Procedure

To download the server package, use the following steps:

  1. From a browser session on the connected machine, go to the following link for information about the Quipucords server package and for the link to download the latest version: Quipucords server package latest version.

  2. To download the server package, click quipucords_server_image.tar.gz.

Building the database container image

The Quipucords server requires a container image for the PostgreSQL database. You must build the image on the connected machine. The type of container image that you need to build depends on the version of your operating system.

Prerequisites

Before you begin, the applicable container software must be installed…​as described in (link to dependencies section…​.)

Procedure

Select one of the following procedures, as appropriate for your operating system:

Building the database container image for RHEL 7 and 8 and CentOS 7 and 8

To build the database container image for for RHEL 7 and 8 and CentOS 7 and 8, use the following steps:

Downloading the command line interface package

The Quipucords command line interface (CLI) package contains a command line interface to interact with the server.

Procedure

Transferring the packages to the offline machine

After you download the Quipucords server and CLI packages and build the PostgreSQL database container image, you must transfer them to the offline machine where Quipucords will be installed.

Procedure

  1. Create a packages directory on the disconnected machine to use for the package transfer, where lib is the library version, either `lib or lib64, and version.release.patch is the complete version number of the qpc-tools RPM package. Use the following command:

    # mkdir -p /usr/_lib_/qpc-tools-_version.release.patch_/install/packages

  1. Move the downloaded packages from the connected machine into the packages directory on the disconnected machine, where pathtopackages is the path to the packages directory. Use the following command:

    # mv /_pathtopackages_/usr/_lib_/qpc-tools-_version.release.patch_/install/packages
